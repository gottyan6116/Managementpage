# 01 — 要件定義・機能一覧

## 1. 利用形態とユーザー

- **オーナー（1名）**: 唯一ログインするユーザー。全データの所有者。Supabase Auth で認証。
- **メンバー（members）**: 「山田太郎」「佐藤花子」「鈴木一郎」「田中美咲」など。
  ログインしない **マスタデータ**。タスクや案件の「担当者」として割り当てる対象。
  - 氏名 / 役職（例: コンサルタント）/ アバター画像 / 表示色 を持つ。
  - オーナー自身も members の 1 レコードとして登録（ヘッダーのプロフィール表示用）。

> この設計により「個人利用のシンプルさ」と「担当者アイコンが多数並ぶ UI」を両立する。
> 将来チーム化する場合は members に `auth_user_id` を追加して拡張できる（`docs/03` 参照）。

## 2. ドメイン用語

| 用語 | 意味 |
|------|------|
| プロジェクト / 案件 (project) | クライアント案件。複数のタスク・マイルストーンを持つ |
| タスク (task) | プロジェクト配下の作業単位。担当者・期限・優先度・ステータスを持つ |
| マイルストーン (milestone) | ガント上の重要日付（◆ で表示） |
| ステータス (status) | 未着手 / 進行中 / 完了 / 保留 / 中止 |
| 優先度 (priority) | 高 / 中 / 低 |
| フェーズ (phase) | 案件の段階（要件定義 / 設計 / 制作 / 開発 / 分析 / 構築 / テスト 等） |
| ボード列 (board_column) | カンバンの列（未着手 / 進行中 / 完了） |

## 3. 機能要件（FR）

### 3.1 共通レイアウト
- FR-L1: 左サイドバー（ロゴ＋7メニュー＋下部プロフィール）。アクティブ項目をハイライト。
- FR-L2: 上部ヘッダー（ハンバーガー、グローバル検索 `⌘K`、通知ベル（バッジ数）、ヘルプ、「+ 新規作成」ドロップダウン）。
- FR-L3: サイドバー折りたたみ（ハンバーガーでアイコンのみ表示に）。

### 3.2 Todo ダッシュボード `/todo`
- FR-T1: KPI カード 4 種（担当プロジェクト数＋前月比＋スパークライン、タスク完了率＋ドーナツ、今週の予定タスク＋棒グラフ、今月の期限タスク＋棒グラフ）。
- FR-T2: 「ガントチャート（今週のプレビュー）」ミニガント。
- FR-T3: 「今後の期限」リスト（色ドット＋案件＋日付＋残り日数）。
- FR-T4: 「タスク一覧」テーブル（タブ: すべて/自分のタスク/期限超過/完了、フィルター、表示設定）。列: タスク名・プロジェクト・担当者・期限・優先度・ステータス。
- FR-T5: 「ボード（タスクボード プレビュー）」3 列カンバンの抜粋。

### 3.3 ガントチャート `/gantt`
- FR-G1: 上部 KPI 4 種（進行中タスク数 / 遅延タスク / 今週のマイルストーン / 稼働率）。
- FR-G2: フィルターバー（プロジェクト / 担当者 / ステータス / 期間 / ビュー / リセット）。
- FR-G3: 左ツリー（プロジェクト→タスク階層、担当者・進捗%・ステータス）。
- FR-G4: 右タイムライン（週グリッド、バー、依存矢印、マイルストーン◆、今日ライン）。
- FR-G5: 「今日にフィット」「ズーム±」「全画面」操作。
- FR-G6: 右ペイン（近日のマイルストーン / プロジェクト進捗サマリー）。
- FR-G7（フェーズ2）: バーのドラッグで期間変更、進捗ハンドル。

### 3.4 担当案件 `/projects`
- FR-P1: 上部 KPI 4 種（担当案件数 / 進行中 / 要注意 / 今月完了予定）。
- FR-P2: タブ（すべて / 進行中 / 完了 / 保留）＋フィルター＋並び替え。
- FR-P3: 案件テーブル（プロジェクト名・クライアント・フェーズ・進捗バー・次回期限＋残り日数・担当者アバター群・優先度・ステータス・⋮メニュー）。
- FR-P4: 右ペイン（要注意案件 / 今週のアクション（チェック可）/ 案件ステータス分布ドーナツ）。

### 3.5 ボード `/board`
- FR-B1: カンバン（未着手 / 進行中 / 完了、各列に件数）。カード = タスク。
- FR-B2: @dnd-kit でカード D&D、列移動でステータス更新。

### 3.6 ドキュメント / ファイル / メモ
- FR-D1: ドキュメント一覧（リッチテキスト or Markdown 編集、フェーズ3で簡易エディタ）。
- FR-F1: ファイル一覧（Supabase Storage、アップロード/プレビュー/ダウンロード）。
- FR-N1: メモ（軽量ノート、ピン留め・色分け）。

### 3.7 横断機能
- FR-X1: グローバル検索（プロジェクト・タスク・ドキュメント横断）。
- FR-X2: 通知（期限接近・遅延を生成、ベルにバッジ）。
- FR-X3: 「+ 新規作成」から プロジェクト/タスク/メモ等を作成。

## 4. 非機能要件（NFR）

- NFR-1 パフォーマンス: 初期表示 LCP < 2.5s。ガントは仮想化不要規模だが 200 タスクで滑らかに。
- NFR-2 アクセシビリティ: WCAG AA。キーボード操作・フォーカスリング・コントラスト遵守。
- NFR-3 レスポンシブ: デスクトップ最適化が主。1280px 基準。タブレットでサイドバー折りたたみ。
- NFR-4 国際化: 文言は日本語。日付は `date-fns` ロケール ja。将来 i18n 化できる構造に。
- NFR-5 セキュリティ: Supabase RLS でオーナーのデータのみ参照可。環境変数で鍵管理。
- NFR-6 保守性: データ取得はフックに集約。マジックナンバー禁止。型は Supabase から生成。

## 5. スコープ外（今回作らない）

- 複数テナント / 課金 / 招待によるメンバーログイン
- ネイティブモバイルアプリ
- 高度な権限管理（RBAC）
- 外部カレンダー連携（将来拡張候補として `docs/05` に記載）
