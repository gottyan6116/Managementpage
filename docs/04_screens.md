> （スクショ再現）から進化したため、本ドキュメントを実装準拠に更新した。

各画面は「レイアウト → 構成要素 → データ → 状態/操作 → コンポーネント分割」で記述。

## 共通: AppShell（全画面の外枠）

```
<AppShell>
  <Sidebar />            // 左 248px(可変)、白ガラス、下部プロフィール
  <div main>
    <Header />           // 64px：☰ / 検索(Ctrl+K/⌘K) / 🔔 / ? / [+ 新規作成 ▼]
    <main>{page}</main>  // padding 24px、max-width 1600px
  </div>
</AppShell>
```

### Sidebar
- 上: ロゴ「ProManage」＋小さく "Work Together, Succeed."。
- メニュー命名は **1対1 を厳守**（同じ語が複数の場所を指してはならない）:
  - ワークスペース: **ホーム** (`/todo`) / ガントチャート (`/gantt`) / **ボード** (`/board`) / 工数 (`/time`)
  - 案件: 担当案件 (`/projects`) / 請求・売上 (`/billing`)
  - ナレッジ: ドキュメント (`/documents`) / メモ (`/notes`)
- アクティブはライトブルーカード化（`sidebar-nav-active`）。
- 下: プロフィールカード（アバター＋氏名/ロール＋ `ChevronDown`）。
- デスクトップ (lg 以上): 折りたたみ 80px（アイコンのみ、ラベルは tooltip）＋幅ドラッグ調整。
- **モバイル (lg 未満): オフキャンバスドロワー**（☰で開く / オーバーレイ・Escape・遷移で閉じる）。
- 状態: `useUiStore().sidebarCollapsed` / `mobileSidebarOpen`（Zustand）。

### Header
- 左: ☰（lg 以上=折りたたみトグル / lg 未満=ドロワーを開く）。
- 中央: 検索インプット（プレースホルダ「プロジェクト、タスク、ドキュメントを検索...」、
  右にショートカット表記。**OS 判定で Windows=Ctrl K / Mac=⌘K**）。
- 右: 通知ベル（赤バッジ件数、role=menu・Escape対応）/ ヘルプ `?` /
  「+ 新規作成」（青ボタン＋ドロップダウン: プロジェクト/タスク/メモ/ドキュメント）。
- ダークモード切替はダークトークン整備まで非表示。

### 共通コンポーネント（`components/shared`）
`StatusBadge` `PriorityBadge` `Avatar` `AvatarGroup` `ProgressBar` `KpiCard` `SectionCard` `DueText` `Donut` `Sparkline` `MiniBarChart`。

## §1. ホーム `/todo`（サマリーダッシュボード）

画面のジョブ: 「今日、何から着手し、何が危ないか」に10秒で答え、そのまま作業に入れること。
セクションは「判断 → 実行 → 見通し」の順に並べる。

### レイアウト
```
[今日のフォーカス 2/3]              [KPI 2x2 1/3]
[タスク一覧 (全幅、既定タブ=自分のタスク)]
[ガントチャート(今週プレビュー) 2/3]  [今後の期限 1/3]
```

### 今日のフォーカス（ヒーローカード）
- 「最優先」1件＋次に迫るタスク4件。
- 最優先の選定は段階ルール: **期限超過 > 今日期限 > 優先度「高」かつ3日以内 > 期限昇順**。
- 「作業を開く」→ `/board?task=<id>`（ボードで該当カードを展開＋中央スクロール）。
- 「完了にする」→ 完了トグル（楽観的更新）。

### KPI カード（2x2）— 全カードがリンクとして生存していること（自己リンク禁止）
| カード | 遷移先 | 0件時 |
|--------|--------|-------|
| 期限超過 | `/todo?tab=overdue` | ニュートラル表示「超過なし」 |
| 今日締切 | `/todo?due=today` | ニュートラル表示「予定なし」 |
| 今週締切 | `/todo?due=week` | ニュートラル表示「予定なし」 |
| 完了タスク | `/todo?tab=done` | （常に件数表示） |
- 0件は「良い0」として警告色を点灯させない（`kpi-neutral`）。

### タスク一覧
- タブ: すべて / 自分のタスク(既定) / 期限超過 / 完了。**タブ状態は URL クエリ `?tab=` が唯一の状態源**
  （`useSearchParams` で読む。ソフトナビゲーションでも確実に切り替わる）。
- `?due=today|week` の期限フィルタはチップ表示＋✕で解除。タブ切替で自動解除。
- 優先度フィルタはドロップダウン（サイクルボタン禁止）。表示密度トグルあり。
- 列: ○(当たり判定36px) / タスク名 / プロジェクト / 担当者 / 期限 / 優先度 / ステータス。
- タブは `role="tablist"` + 矢印キー移動。
- フッタ: 「残り N 件を表示 ⌄」（件数を明示）。

### ガントチャート（今週のプレビュー）
- **表示範囲は今日±2週にクランプ**し、範囲に重なる行だけ描画（全期間・全行の描画は禁止）。
- 週ヘッダーはボディの横スクロールに**常に追従**（日付とバーの対応を保証）。
- 初期表示で今日が中央に来るよう自動スクロール。今日ラインに「今日 M/d」バッジ。
- 右上に「ガントチャートへ ＞」リンク（`/gantt`）。実体は `<GanttChart variant="preview" />`。

### 今後の期限（右下）
- リスト各行: 色ドット＋タスク名＋（プロジェクト名）＋右に日付＋残り日数。
- **マイルストーンは ◆ アイコン＋「MS」チップで種別を明示**（タスクとの誤読防止）。
- 右上「ガントで見る」（ラベルは行き先と一致させる）。
- データ: `listUpcoming` = 未完了 milestones + 未完了 tasks.due_date 近い順。

### 操作 / UX
- タスク行 ○ クリックで完了トグル（楽観的更新→トースト）。
- 「今日」は `appToday()`（JST の暦日）を使う。固定日付のハードコード禁止。

## §2. ガントチャート `/gantt`（スクショ2枚目・最重要）

### レイアウト
```
タイトル「ガントチャート」＋サブ「全プロジェクトの進行状況と依存関係を可視化」   [KPI ×4]
[フィルターバー]
[左:タスクツリー表][右:タイムライン]            [右ペイン: 近日マイルストーン / 進捗サマリー]
```

### KPI（4枚・右上）
進行中タスク数 42（全体58%）/ 遅延タスク 8（全体11%・赤）/ 今週のマイルストーン 6（完了予定）/ 稼働率 78%（最適水準・ドーナツ）。

### フィルターバー
プロジェクト すべて▼ / 担当者 すべて▼ / ステータス すべて▼ / 期間 2025/05/01–2025/05/31 / ビュー / リセット。

### 左ツリー表（列）
タスク名（▼で開閉、階層インデント）/ 担当者(アバター) / 進捗(%) / ステータス(バッジ) / 「+」列追加。
- 親 = プロジェクト行（色ドット＋太字＋全体%）。子 = タスク（`>` インデント）。

### 右タイムライン（描画仕様）
- 列ヘッダ: 週区切り（4/28–5/4, 5/5–5/11 …）＋下に曜日(月火水木金土日)。
- 1日 = 列幅 `dayWidth`（既定 28px、ズームで 16〜48px）。
- バー: `left = (start - rangeStart)*dayWidth`、`width = (due - start + 1)*dayWidth`。色=プロジェクト色。
- マイルストーン: ◆を due 位置に。バー右端に終了日ラベル。
- 依存矢印: `task_dependencies` を辿り predecessor 終端→successor 始端へL字コネクタ。
- 今日ライン: `current_date` 位置に青縦線＋「今日 5/16」バッジ。
- 「今日にフィット」=スクロールを今日中心へ。ズーム± = dayWidth 変更。全画面=パネル拡大。

### 実装メモ（重要）
- ガントは **左右2スクロール領域を縦同期**（行高さを完全一致させる）。
- 行データ型: `GanttRow { id, type:'project'|'task', depth, label, assignee, progress, status, bar:{start,due,color}, milestone? }`。
- まず CSS Grid + 絶対配置バーで描画。仮想化は後回し（200行までは不要）。
- フェーズ2: バー左右端ドラッグで `start/due` 更新、本体ドラッグで平行移動、進捗ハンドル。

### 右ペイン
- 近日のマイルストーン（フラグ＋タイトル＋プロジェクト＋日付＋「今日/明日/N日後」）。
- プロジェクト進捗サマリー（色ドット＋名＋進捗バー＋%、最下部に全体平均）。

## §3. 担当案件 `/projects`（スクショ3枚目）

### レイアウト
```
タイトル「担当案件」＋サブ「自分が担当する案件の状況を一覧で管理」
[KPI ×4]
[左: タブ＋テーブル 2fr]      [右ペイン 1fr: 要注意案件 / 今週のアクション / ステータス分布]
```

### KPI（4枚）
担当案件数 12件（前月比+2件・スパークライン）/ 進行中 8件（全体66%・ドーナツ）/ 要注意 2件（前月比-1件・棒）/ 今月完了予定 3件（前月比+1件・棒）。

### テーブル
- タブ: すべて / 進行中 / 完了 / 保留。右: フィルター / 並び替え（次回期限が近い順▼）。
- 列: プロジェクト名+クライアント / フェーズ(バッジ) / 進捗(バー+%) / 次回期限(日付+残り日数) / 担当者(アバター群+N) / 優先度 / ステータス / ⋮。
- 行例:
  - 新規業務改革プロジェクト・グローバルホールディングス・要件定義・72%・5/20(火)3日後・高・進行中
  - 営業資料リニューアル・サクセスパートナーズ・制作・90%・5/16(金)今日・中・最終確認
  - 問い合わせフォーム改善・ビジネスフロンティア・テスト・100%・5/09(金)完了・低・完了
- フッタ「全12件を表示 ⌄」。

### 右ペイン
- 要注意案件（⚠＋案件名＋「次回期限が2日後」＋日付。赤系）。
- 今週のアクション（チェックボックス＋内容＋日付。チェックでトグル＝`actions.is_done`）。
- 案件ステータス分布（ドーナツ中心「12件」＋凡例: 進行中8(66%)/最終確認1(8%)/完了2(16%)/保留1(8%)/中止0(0%)）。右上に期間セレクタ「今月▼」。

### 操作
- 行クリックで案件詳細（フェーズ2: `/projects/[id]`）。⋮ で 編集/複製/削除。

## §4. ボード `/board`

- 列 = `board_columns`（未着手/進行中/完了）。各列ヘッダに件数＋「＋」。
- カード = タスク（タイトル＋プロジェクト＋担当アバター＋優先度ドット）。
- @dnd-kit で D&D。列間移動 → `tasks.status` と `board_column_id` 更新（楽観的）。
- 列内並べ替え → `board_position` 更新。

---

## §5. ドキュメント `/documents`
- 一覧（タイトル/プロジェクト/更新日）＋「新規ドキュメント」。
- 詳細は Markdown エディタ（フェーズ3、`@uiw/react-md-editor` 等の軽量採用可）。

## §6. ファイル `/files`
- グリッド/リスト切替。アップロード（Supabase Storage バケット `files`）。
- カード: アイコン/サムネ＋名前＋サイズ＋日付。プレビュー/ダウンロード/削除。

## §7. メモ `/notes`
- カード型（色付き付箋）。ピン留めを上部に。クリックでインライン編集。新規はマソンリー追加。

---

## 空状態・ローディング・エラー（全画面共通 UX）
- ローディング: スケルトン（カード形状に合わせる）。
- 空状態: イラスト＋一言＋主要 CTA（例「最初のプロジェクトを作成」）。
- エラー: インラインの再試行ボタン＋トースト。ネットワーク断は楽観的更新をロールバック。
